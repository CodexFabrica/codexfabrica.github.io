## Product Portfolio

### 1. Feder

**Status**: Beta Release  
**Distribution Model**: Freeware  
**Product Type**: Web Application  
**Repository**: https://github.com/CodexFabrica/Feder  
**Documentation**: https://codexfabrica.github.io/products/feder/

#### Description
Feder represents an open-source, Markdown-centric workspace architected specifically for writers, researchers, and technical documentation specialists. It functions as a serverless, privacy-respecting alternative to established platforms such as Overleaf, eschewing the inherent complexity of LaTeX typesetting systems in favor of a streamlined Markdown paradigm.

#### Key Characteristics
- **Markdown-First Methodology**: Leverages clean, human-readable Markdown syntax as the primary content authoring format
- **Real-Time Preview Rendering**: Provides instantaneous visual feedback through live document preview functionality
- **Version Control Integration**: Maintains compatibility with Git-based workflows through plain-text file architecture
- **Minimal Configuration Requirements**: Enables rapid deployment with minimal setup prerequisites
- **Open Source Licensing**: Freely available for use, modification, and community contribution

#### Target Audience
Academic researchers, technical writers, documentation specialists, creative content authors, and professionals seeking alternatives to complex typesetting environments.

#### Technical Prerequisites
- Node.js (version 16 or higher)
- npm package manager
- Git version control system

#### Use Cases

## Research Initiatives

### Cognitive Code Annotation

**Research Domain**: Developer Tooling and Collaborative Documentation Systems  
**Status**: Active Research  
**Documentation**: https://codexfabrica.github.io/research/example-project/

#### Research Objectives
This initiative investigates the feasibility and implementation methodologies for a dual-pane documentation paradigm that integrates contextual annotations directly alongside code segments without disrupting script execution or version control workflows.

#### Conceptual Framework
Traditional inline comments exhibit limitations in their linear organizational structure, often cluttering logical flow and reducing code readability. The Cognitive Code Annotation research proposes an alternative architecture featuring:

1. **Primary Code Pane**: Contains the executable source code in its pristine, uncommented state
2. **Parallel Annotation Pane**: Displays contextual commentary, explanations, and collaborative feedback anchored to specific code regions (individual lines, functions, or algorithmic blocks)

#### The "Anti-Notebook" Philosophy
While computational notebooks (e.g., Jupyter) have popularized the interleaving of executable code and narrative prose, they frequently disrupt conventional IDE workflows and complicate version control integration. This research endeavors to preserve the integrity of pure code scripts while rendering supplementary annotations in dedicated visual spaces within IDEs or documentation viewers.

#### Technical Challenges Being Addressed
- **Region Identification Mechanisms**: Developing sophisticated methods for delineating code regions (AST-based parsing versus line-range specifications)
- **Threaded Communication Infrastructure**: Implementing hierarchical feedback structures within annotation panes
- **Cross-Platform Portability**: Ensuring annotation persistence and accessibility across heterogeneous development environments (VS Code, JetBrains IDEs, Feder, etc.)

#### Anticipated Outcomes
A standardized annotation format that enhances code comprehension, facilitates asynchronous code review processes, and preserves the execution integrity of source code while enriching it with contextual human communication.
